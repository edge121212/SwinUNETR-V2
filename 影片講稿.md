## 0:00–0:45　開場與論文簡介
〔畫面：論文標題頁 / Fig.2 架構圖〕

大家好，這次我們重現的論文是 **SwinUNETR-V2**，這是一篇 3D 醫學影像分割的論文。
它的核心想法是：在原本的 SwinUNETR（Swin Transformer 當 encoder 的 U 型網路）的基礎上，
**在每個 stage 的開頭加上一個殘差卷積（ResConv）block**，用卷積先「整理」特徵，再丟給後面的
Swin Transformer block。作者發現這個簡單的改動，不需要針對每個資料集大幅調參，就能在多個
benchmark（WORD、FLARE2021、MSD prostate/lung/pancreas）達到 SOTA。

我們選擇用論文中的 **MSD Task05 Prostate** 資料集來重現它的 5-fold 交叉驗證結果。

## 0:45–2:15　資料集介紹
〔畫面：dataset.json 內容 + 一張 MRI 切片〕

先簡單說一下 **MSD 挑戰賽**：它的目標是測試一套演算法在 **10 個不同器官分割任務**上的
**泛化能力**——重點是要能「**不靠人工針對個別任務調參**」，就獨立把每個任務解好
（可以用單一模型、ensemble、架構搜尋等任何技術）。我們選的 **Task05 Prostate** 就是其中之一。



資料集是 **Medical Segmentation Decathlon (MSD) 的 Task05 Prostate**：

- **內容**：前列腺 MRI，共 **32 筆有標註**的訓練資料（另有 16 筆無標註測試，不使用）。
- **兩種模態（in_channels = 2）**：`T2` 加權影像 + `ADC` 圖，兩個 channel 疊在一起輸入。
  - **T2（T2 加權）**：最常見的 MRI，看的是**解剖結構**，器官輪廓、PZ/TZ 的形狀都清楚。
  - **ADC（表觀擴散係數圖）**：由擴散影像算出來的圖，反映**組織特性**（水分子擴散自由度）；
    惡性、細胞密集的組織 ADC 值偏低。
  - 兩者**互補**：T2 給「形狀/邊界」、ADC 給「組織特性」，合起來讓模型分割得更準
    （就像彩色圖的 RGB 是 3 個 channel，這裡是 T2/ADC 兩個 channel）。
- **三個分割類別（out_channels = 3）**：
  - `0` = 背景
  - `1` = **PZ**（周邊區 Peripheral Zone）
  - `2` = **TZ**（移行區 Transition Zone）
- **評估指標**：Dice score，分別算 PZ、TZ，再取平均（Mean Dice）。
- **驗證方式**：跟論文一樣做 **5-fold cross-validation**，最後取 5 個 fold 的平均。

> 講解重點：前列腺只有 32 筆資料、又是兩種模態，資料量小是這個 task Dice 偏低、
> 而且 fold 之間變異較大的主要原因。

## 2:15–5:30　程式碼介紹（每塊在做什麼）
〔畫面：依序開啟各檔案，捲動到對應區塊〕

整個專案分成「資料 → 模型 → 訓練 → 測試」四塊。

### (1) 資料載入與切分 — `utils/data_utils.py` 的 `get_loader()`
- 讀 `dataset.json` 的 32 筆訓練資料，用**固定亂數種子 42** 洗牌，確保每次切分一致。
- 把資料切成 10 塊，依 `--fold`（0~4）決定哪幾塊當 test / val / train，達成 **5-fold**。
- **前處理 transforms**（重點唸這幾個就好）：
  - `Orientationd` 統一方向、`Spacingd` 重採樣到 1×1×1 mm；
  - `NormalizeIntensityd` 做強度正規化（MRI 用 z-score，CT task 會多做百分位裁切）；
  - `RandCropByPosNegLabeld` 隨機裁出 `64×64×64` 的 patch（正/負樣本各半）；
  - 之後是一連串資料增強：隨機翻轉、旋轉、強度縮放/平移、高斯模糊/雜訊、對比調整。
- 驗證/測試則不做增強，直接整張影像丟進去。

### (2) 模型 — `main.py` 的 `SwinUNETR(...)`（第 153 行附近）
- 直接用 MONAI 內建的 `SwinUNETR`：Swin Transformer encoder + CNN decoder 的 U 型架構，
  `feature_size=48`、`spatial_dims=3`。
- 損失函數用 `DiceCELoss`（Dice loss + Cross-Entropy 各半），優化器用 `AdamW`，
  學習率排程用 **warmup + cosine annealing**。
- 推論時用 **sliding window inference**（`roi=64³`，overlap=0.5）把整張影像滑窗拼接出來。


### (3) 訓練迴圈 — `trainer.py`
- `train_epoch()`：標準訓練流程，搭配 **AMP 混合精度**（GradScaler）加速、省顯存。
- `val_epoch()`：每 `--val_every` 個 epoch 用 `DiceMetric` 在驗證集算一次 Dice。
- `run_training()`：主迴圈，記錄 loss/acc 到 TensorBoard，
  只要驗證 Dice 創新高就把權重存成 `model.pt`，每次驗證也會存 `model_final.pt`（可斷點續訓）。

### (4) 測試評估 — `test.py`
- 載入訓練好的權重，對 test fold 每個 case 做 sliding window 推論。
- 把預測 resample 回原始解析度，對每個類別算 Dice，
  最後印出 `Overall PZ Dice`、`Overall TZ Dice`、`Overall Mean Dice`。

### (5) 5-fold 彙整 — `utils/aggregate_kfold.py`
- 掃描 5 個 fold 的 test log，把 PZ / TZ / Mean 取平均與標準差印出來，就是最終成績。

## 5:30–6:45　怎麼執行你的程式碼
〔畫面：終端機實際打指令（或事先錄好的操作）〕

我們用 `run.bat`（Windows）/ `run.sh`（Linux）把流程包好，分四個動作：

```bash
# 1. 下載資料集
run.bat download Prostate

# 2. 單一 fold 訓練（fold 0）
run.bat train Prostate

# 3. 單一 fold 測試
run.bat test Prostate

# 4. 一鍵跑完整 5-fold（訓練 + 測試 + 自動彙整）
run.bat kfold Prostate 500
```

- `kfold` 會對 fold 0~4 依序「訓練 → 測試」，已存在 `model_final.pt` 的 fold 會自動跳過，
  所以**中途關機也能從斷點接著跑**。
- 每個 fold 的測試結果存到 `runs/prostate_fold{N}_test.log`。
- 全部跑完會自動呼叫 `aggregate_kfold.py`，印出 5-fold 平均。

> 環境：單張 GPU、`roi=64³`、`max_epochs=500`、`val_every=25`。

## 6:45–9:15　復現結果 vs 原論文
〔畫面：下面這張比較表 + TensorBoard 曲線（可選）〕

我們的 5-fold 平均 Dice 與論文 Table 4 的比較：

| 指標 | 原論文 SwinUNETR-V2 | 原論文 SwinUNETR | **我們的復現** | 與 V2 差距 |
|---|---|---|---|---|
| PZ（周邊區）| 0.6353 | 0.6167 | **0.6025** | −0.033 |
| TZ（移行區）| 0.8457 | 0.8498 | **0.7846** | −0.061 |
| Mean（平均）| 0.7405 | 0.7332 | **0.6936** | −0.047 |

**結論：三項指標都落在原論文約 3~6 個百分點以內，成功復現了論文的量級。**

差距的可能原因（建議逐點說明）：
1. **訓練量較少**：我們用 `roi=64³`、單卡、約 500 epoch；論文用更大的 patch、
   並訓練到約 **40k iterations**、多卡，模型收斂得更充分。
2. **未開啟 V2 結構**：如前述，我們跑的是原版 SwinUNETR backbone，
   少了 V2 的 stage-wise ResConv，這會讓表現略低於 V2 那一列。
3. **資料量小、fold 變異大**：Prostate 只有 32 筆，5-fold 之間本來就有較大波動
   （我們的標準差約 0.06~0.08），單次復現落在這個範圍內屬正常。
4. **隨機性**：資料切分、增強、初始化的隨機種子與論文不完全相同。

整體而言，TZ 因為區域大、邊界清楚，Dice 最高；PZ 區域小、邊界模糊，最難分割，
這個**類別間的難易趨勢和論文完全一致**，代表復現是合理且正確的。
